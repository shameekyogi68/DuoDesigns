# 🌐 Domain & DNS Configuration

Duo Designs uses a specialized subdomain structure to separate user concerns and improve security.

## 🏛️ Domain Hierarchy

| Hostname | Target Service | Purpose |
| :--- | :--- | :--- |
| `duodesigns.in` | Vercel (Production) | Primary Storefront |
| `admin.duodesigns.in` | Vercel (Production) | Platform Operations |
| `api.duodesigns.in` | Render (Backend) | Core Logic Engine |
| `staging.duodesigns.in`| Vercel (Preview) | Public Testing/QA |

---

## 🛠️ DNS Pointer Settings (A Records / CNAME)

### Customer App
- **CNAME:** `cname.vercel-dns.com`
- **SSL:** Auto-generated.

### Admin App
- **CNAME:** `cname.vercel-dns.com`
- **SSL:** Auto-generated.

### Backend API
- **CNAME:** `onrender.com` (Targeting the service slug).
- **SSL:** Auto-generated by Render.

---

## 🛡️ Email Domain Config (SPF/DKIM)
Since we send transactional emails from `hello@duodesigns.in`, the following DNS records are set to ensure we don't land in Spam:
- **SPF:** `v=spf1 include:_spf.google.com ~all`
- **DKIM:** Key provided by Google Workspace.

## 🕒 TTL Strategy
During major migrations, TTL is set to **300 (5 mins)**. Once stable, it is updated to **3600 (1 hour)** to improve performance and reduce DNS lookups.
